The Google File System (GFS) is a scalable distributed file system for large
distributed data-intensive applications. It provides fault tolerance while
running on inexpensive commodity hardware, and it delivers high aggregate
performance to a large number of clients.

This sample file is intentionally longer than one kilobyte so that the client
splits it into several 1 KB chunks. Each chunk is then replicated across more
than one storage server, and the naming server records where every replica
lives. When you read the file back, the client fetches each chunk from any
available replica and reassembles the original content.

Partitioning spreads the chunks across servers so that no single machine holds
the whole dataset; replication keeps more than one copy of each chunk so the
system survives the loss of a server; and the separation between the naming
(metadata) layer and the storage layer keeps the design clean and easy to
reason about. Those three ideas are the heart of this project.
